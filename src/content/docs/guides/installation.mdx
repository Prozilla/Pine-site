The Pine engine is distributed using [GitHub Packages](https://github.com/Prozilla/Pine/packages/2343840). The [GitHub Packages docs](https://docs.github.com/en/packages) contain more information about how it works.

:::note

Some examples below might use an older version of Pine. Make sure you replace the version number with the latest version of Pine, or with whatever version you want to use.

<LinkButton href="https://github.com/Prozilla/Pine/releases" variant="secondary" icon="external">All releases</LinkButton>
<LinkButton href="https://github.com/Prozilla/Pine/releases/latest" variant="secondary" icon="external">Latest release</LinkButton>

:::

## Option 1: Use boilerplate project setup

This is the same as [option 2](#option-2-install-using-gradle), but with most of the work already done for you.

<Steps>

1. Fork the [Pine-boilerplate repository](https://github.com/Prozilla/Pine-boilerplate)
2. Follow the setup guide in the README file
3. Change the package name to something that suits your project (Optional)

</Steps>

## Option 2: Install using Gradle

<Steps>

1. In your `build.gradle` file, add the following lines:

	<Tabs>
		<TabItem label="Groovy">
			<Code code={`
		repositories {
			mavenCentral()
			maven {
				url "https://repo.gradle.org/gradle/libs-releases"
			}
			maven {
				url = uri("https://maven.pkg.github.com/prozilla/pine")
				credentials {
					username = project.findProperty("gpr.user") ?: System.getenv("USERNAME")
					password = project.findProperty("gpr.token") ?: System.getenv("TOKEN")
				}
			}
		}

		dependencies {
			implementation "dev.prozilla:pine:${PINE_VERSION}"
		}
			`} title="build.gradle" lang="groovy" />
		</TabItem>
		<TabItem label="Kotlin">
			<Code code={`
		repositories {
			mavenCentral()
			maven {
				url = uri("https://repo.gradle.org/gradle/libs-releases")
			}
			maven {
				url = uri("https://maven.pkg.github.com/prozilla/pine")
				credentials {
					username = findProperty("gpr.user") as String? ?: System.getenv("USERNAME")
					password = findProperty("gpr.token") as String? ?: System.getenv("TOKEN")
				}
			}
		}

		dependencies {
			implementation("dev.prozilla:pine:${PINE_VERSION}")
		}
			`} title="build.gradle.kts" lang="kotlin" />
		</TabItem>
	</Tabs>

2. Create a `gradle.properties` file or a `.env` file with your environment variables. Replace `YOUR_USERNAME` and `YOUR_TOKEN` with your GitHub username and token respectively.

	<Tabs>
		<TabItem label="gradle.properties">
			<Code code={"gpr.user=YOUR_USERNAME\ngpr.token=YOUR_TOKEN"} title="gradle.properties" lang="properties" />
		</TabItem>
		<TabItem label=".env">
			<Code code={"USERNAME=YOUR_USERNAME\nTOKEN=YOUR_TOKEN"} title=".env" lang="properties" />
		</TabItem>
	</Tabs>

	Make sure to add this file to `.gitignore` so it remains private.

</Steps>

## Option 3: Install using Maven

<Steps>

1. In your `pom.xml` file, add the following lines:

	<Code code={`
		<repositories>
			<repository>
				<id>gradle-releases</id>
				<url>https://repo.gradle.org/gradle/libs-releases</url>
			</repository>

			<repository>
				<id>pine-github</id>
				<url>https://maven.pkg.github.com/prozilla/pine</url>
			</repository>
		</repositories>

		<dependencies>
			<dependency>
				<groupId>dev.prozilla</groupId>
				<artifactId>pine</artifactId>
				<version>${PINE_VERSION}</version>
			</dependency>
		</dependencies>
	`} title="pom.xml" lang="xml" />

2. Create a global `settings.xml` file in your Maven configuration folder (`.m2`) that stores your GitHub credentials. Replace `YOUR_USERNAME` and `YOUR_TOKEN` with your GitHub username and token respectively.

	<Code code={`
		<settings>
			<servers>
				<server>
					<id>pine-github</id>
					<username>YOUR_USERNAME</username>
					<password>YOUR_TOKEN</password>
				</server>
			</servers>
		</settings>
	`} title=".m2/settings.xml" lang="xml"/>

</Steps>

## Option 4: Using JARs

It is possible to use Pine by downloading the JARs of Pine and its dependencies and adding them to the classpath.
Sources and Javadoc JARs are also available.

This can also be used to install unreleased versions of Pine, like the [development branch](https://github.com/Prozilla/Pine/tree/dev) or a fork, by building Pine from source.

<Steps>

1. Go to the [Releases page](https://github.com/Prozilla/Pine/releases) and download the JAR of the version you want to use, or build Pine from source.
2. Download the JARs of each dependency:

	- [LWJGL](https://www.lwjgl.org/download)
		- Core
		- GLFW
		- OpenGL
		- OpenAL
		- stb
		- Natives (Determined by the target platform)
	- [Jackson](https://repo1.maven.org/maven2/com/fasterxml/jackson/core/)
		- Core
		- Databind
		- Annotations

	<br/>

3. Create a `libs` folder and move the JARs inside this folder
4. Add all JARs to your classpath

</Steps>

### Hybrid usage

When using a build system, it is possible to only download the Pine JAR manually and to import the other dependencies using said build system.

<Tabs>
	<TabItem label="Gradle">
		<Code code={`
	dependencies {
		implementation files("libs/pine-engine-${PINE_VERSION}.jar")

		// LWJGL
		// Use https://www.lwjgl.org/customize to install the required LWJGL packages

		// Other Pine dependencies
		implementation "com.fasterxml.jackson.core:jackson-databind:2.18.1"
		implementation "org.gradle:gradle-tooling-api:8.8"
		runtimeOnly "org.slf4j:slf4j-simple:2.0.16"
	}
		`} title="build.gradle" lang="groovy" />
	</TabItem>
</Tabs>

## Minimal example

The following code can be used to quickly verify whether the installation of Pine was succesful or not. If no exception is thrown and a blank window opens, that means everything needed to run a Pine application is installed correctly.

```java
public class Main {

	public static void main(String[] args) {
		new ApplicationBuilder().build().run();
	}

}
```
